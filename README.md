# sqlc-gen-bulk-insert

A [sqlc](https://sqlc.dev) **process plugin** that generates bulk-insert helper
functions for **MySQL / MariaDB** using the `VALUES (?, ?), (?, ?)` approach.

Instead of running `N` individual INSERT statements in a loop:

```go
for _, u := range users {
    err = q.InsertUser(ctx, InsertUserParams{Name: u.Name, Email: u.Email})
}
```

the generated helper builds a single batched statement:

```sql
INSERT INTO users (name, email) VALUES (?, ?), (?, ?), (?, ?)
```

This is significantly faster for large data sets and avoids N round-trip
overhead.

---

## Requirements

| Dependency | Version |
|---|---|
| Go | ≥ 1.21 |
| sqlc | ≥ 1.24 (tested with 1.30.0) |
| MySQL / MariaDB | any version supporting multi-row `INSERT … VALUES` |

---

## Installation

```bash
go install github.com/puchentw/sqlc-gen-bulk-insert@latest
```

The `sqlc-gen-bulk-insert` binary must be on your `$PATH` when you run
`sqlc generate`.

---

## Configuration

Add the plugin to your `sqlc.yaml` and point the `codegen` output at the
**same directory** as your regular Go code-gen output so both share the same
package:

```yaml
version: "2"

plugins:
  - name: sqlc-gen-bulk-insert
    process:
      cmd: sqlc-gen-bulk-insert   # binary on $PATH

sql:
  - engine: "mysql"
    queries: "query.sql"
    schema:  "schema.sql"

    gen:
      go:
        package: "db"
        out:     "db"

    codegen:
      - plugin:  sqlc-gen-bulk-insert
        out:     "db"       # same output dir as gen.go
        options:
          package: "db"     # must match gen.go.package
```

### Plugin options

| Option | Type | Default | Description |
|---|---|---|---|
| `package` | string | **required** | Go package name for the generated file(s) |
| `out_filename` | string | `bulk_insert.go` | Output file name — only used when `split_by` is `"single"` |
| `split_by` | string | `"single"` | Controls how many files are produced (see below) |
| `emit_interface` | bool | `false` | Also emit an interface declaring every bulk method (see [Generated interface](#generated-interface-for-mocking)) |
| `interface_name` | string | `BulkQuerier` | Name of the emitted interface — only used when `emit_interface` is `true` |

### `split_by` modes

| Value | Output | File naming |
|---|---|---|
| `"single"` *(default)* | One file for all bulk functions | `out_filename` (`bulk_insert.go`) |
| `"file"` | One file per source `.sql` file | `bulk_<sqlfile>.go` |
| `"query"` | One file per generated function | `bulk_<function_snake>.go` |

**`"file"` example** — queries from `users.sql` and `products.sql` produce:

```
db/
├── bulk_users.go
└── bulk_products.go
```

**`"query"` example** — `InsertUser` and `InsertProduct` produce:

```
db/
├── bulk_insert_user.go
└── bulk_insert_product.go
```

Full `sqlc.yaml` example using `split_by: "file"`:

```yaml
codegen:
  - plugin:  sqlc-gen-bulk-insert
    out:     "db"
    options:
      package:  "db"
      split_by: "file"
```

## Which queries are processed?

The plugin generates a `Bulk<QueryName>` function for every query that satisfies
**all** of:

1. It is an `INSERT` statement (sqlc sets `InsertIntoTable` internally).
2. Its command annotation is `:exec`, `:execresult`, or `:execrows`.
3. It has at least one `?` parameter.

SELECT, UPDATE, DELETE queries and parameter-less INSERTs are silently skipped.

## Generated code

### Multi-parameter query

```sql
-- name: InsertUser :exec
INSERT INTO users (name, email) VALUES (?, ?);
```

sqlc's regular code-gen produces `InsertUserParams` and `InsertUser`.
This plugin **additionally** generates:

```go
// Code generated by sqlc-gen-bulk-insert. DO NOT EDIT.

package db

import (
    "context"
    "strings"
)

const bulkInsertUserPrefix = `INSERT INTO users (name, email) VALUES `

// BulkInsertUser performs a bulk INSERT of multiple rows in a single statement.
// Passing an empty slice is a no-op.
func (q *Queries) BulkInsertUser(ctx context.Context, args []InsertUserParams) error {
    if len(args) == 0 {
        return nil
    }
    valueStrings := make([]string, 0, len(args))
    valueArgs := make([]any, 0, len(args)*2)
    for _, arg := range args {
        valueStrings = append(valueStrings, "(?, ?)")
        valueArgs = append(valueArgs, arg.Name, arg.Email)
    }
    query := bulkInsertUserPrefix + strings.Join(valueStrings, ", ")
    _, err := q.db.ExecContext(ctx, query, valueArgs...)
    return err
}
```

### Single-parameter query

For queries with exactly one parameter, the function takes a `[]<GoType>` slice
instead of a `[]<QueryName>Params` struct slice:

```sql
-- name: InsertAuditLog :exec
INSERT INTO audit_log (user_id) VALUES (?);
```

```go
func (q *Queries) BulkInsertAuditLog(ctx context.Context, args []int64) error { … }
```

### Generated interface (for mocking)

Set `emit_interface: true` to additionally generate a single interface that
declares every bulk method. This is handy for mocking your data layer with
[gomock/mockgen](https://github.com/uber-go/mock),
[mockery](https://github.com/vektra/mockery), or similar tools.

```yaml
codegen:
  - plugin:  sqlc-gen-bulk-insert
    out:     "db"
    options:
      package:        "db"
      emit_interface: true
      interface_name: "BulkQuerier"   # optional, this is the default
```

The interface is always written to its own file
(`<interface_name_snake_case>.go`, e.g. `bulk_querier.go`) **regardless of
`split_by`**, so all methods are aggregated in one place:

```go
// Code generated by sqlc-gen-bulk-insert. DO NOT EDIT.

package db

import (
    "context"
)

// BulkQuerier declares every bulk-insert method generated by
// sqlc-gen-bulk-insert. It is satisfied by the *Queries type and is intended
// for use with mock libraries (gomock/mockgen, mockery, …).
type BulkQuerier interface {
    BulkInsertUser(ctx context.Context, args []InsertUserParams) error
    BulkInsertAuditLog(ctx context.Context, args []int64) error
}
```

Generate a mock against it, for example with `mockgen`:

```bash
mockgen -source=db/bulk_querier.go -destination=db/mock_bulk_querier.go -package=db
```

## MySQL type → Go type mapping

| MySQL type | Go type (NOT NULL) | Go type (nullable) |
|---|---|---|
| `tinyint(1)` / `bool` / `boolean` | `bool` | `sql.NullBool` |
| `tinyint` | `int8` | `sql.NullInt16` |
| `smallint` / `year` | `int16` | `sql.NullInt16` |
| `mediumint` / `int` / `integer` | `int32` | `sql.NullInt32` |
| `bigint` | `int64` | `sql.NullInt64` |
| `float` | `float32` | `sql.NullFloat64` |
| `double` / `real` | `float64` | `sql.NullFloat64` |
| `decimal` / `numeric` | `string` | `sql.NullString` |
| `char` / `varchar` / `text*` | `string` | `sql.NullString` |
| `binary` / `varbinary` / `blob*` | `[]byte` | `[]byte` |
| `date` / `datetime` / `timestamp` | `time.Time` | `sql.NullTime` |
| `json` | `json.RawMessage` | `json.RawMessage` |
| unsigned modifier | `u`-prefixed type (e.g. `uint64`) | — |

> **Note for multi-parameter queries:** the column types above appear inside the
> sqlc-generated `<QueryName>Params` struct, not in the bulk file itself. Extra
> imports (`time`, `encoding/json`, `database/sql`) are only added when a
> single-parameter query uses one of those types directly in the function
> signature.

## Caveats

* **MySQL / MariaDB only.** The plugin uses `?` positional placeholders. It is
  not suitable for PostgreSQL (`$1`, `$2`, …) or named parameters.
* **`q.db` field.** The generated functions call `q.db.ExecContext` directly,
  which requires that your `Queries` struct exposes the standard unexported `db`
  field emitted by `sqlc-gen-go`. Custom db-layer wrappers may need adaptation.
* **`max_allowed_packet`.** Very large batch sizes can exceed MySQL's packet
  limit. Consider chunking on the caller side when inserting thousands of rows.
* **No `Querier` interface update.** The bulk functions are not added to sqlc's
  generated `Querier` interface. Enable [`emit_interface`](#generated-interface-for-mocking)
  to generate a standalone `BulkQuerier` interface instead of hand-writing one.

## Changelog

### v0.0.1 — initial release

* Generate `Bulk<QueryName>` functions for `INSERT` + `:exec` / `:execresult` /
  `:execrows` queries.
* Multi-parameter queries use the existing `<QueryName>Params` struct generated
  by sqlc.
* Single-parameter queries use the mapped Go type directly (`[]int64`,
  `[]string`, …).
* Full MySQL / MariaDB type → Go type mapping including unsigned variants and
  nullable (`sql.NullXxx`) types.
* `split_by` option: `"single"` (default), `"file"`, `"query"`.
* `emit_interface` option: optionally generate a standalone interface
  (`BulkQuerier` by default, configurable via `interface_name`) listing every
  bulk method, for use with mock libraries.

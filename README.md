# sqlc-gen-bulk-insert

A [sqlc](https://sqlc.dev) **process plugin** that generates bulk-insert helper
functions for **MySQL / MariaDB** using the `VALUES (?, ?), (?, ?)` approach.

Instead of running `N` individual INSERT statements in a loop:

```go
for _, u := range users {
    err = q.InsertUser(ctx, InsertUserParams{Name: u.Name, Email: u.Email})
}
```

the generated helper builds a single batched statement:

```sql
INSERT INTO users (name, email) VALUES (?, ?), (?, ?), (?, ?)
```

This is significantly faster for large data sets and avoids round-trip overhead.

---

## Requirements

| Dependency | Version |
|---|---|
| Go  | ≥ 1.21 |
| sqlc | ≥ 1.24 (tested with 1.30.0) |
| MySQL / MariaDB | any version supporting multi-row `INSERT … VALUES` |

---

## Installation

```bash
go install github.com/puchentw/sqlc-gen-bulk-insert@latest
```

The `sqlc-gen-bulk-insert` binary must be on your `$PATH` when you run
`sqlc generate`.

---

## Configuration

Add the plugin to your `sqlc.yaml` and point the `codegen` output at the
**same directory** as your regular Go code-gen output so both share the
same package:

```yaml
version: "2"

plugins:
  - name: sqlc-gen-bulk-insert
    process:
      cmd: sqlc-gen-bulk-insert   # binary on $PATH

sql:
  - engine: "mysql"
    queries: "query.sql"
    schema:  "schema.sql"

    gen:
      go:
        package: "db"
        out:     "db"

    codegen:
      - plugin:  sqlc-gen-bulk-insert
        out:     "db"            # same output dir as gen.go
        options:
          package:      "db"    # must match gen.go.package
          out_filename: "bulk_insert.go"   # optional, default: bulk_insert.go
```

### Plugin options

| Option | Type | Default | Description |
|---|---|---|---|
| `package` | string | **required** | Go package name for the generated file |
| `out_filename` | string | `bulk_insert.go` | File name written to the output directory |

## Which queries are processed?

The plugin generates a `Bulk<QueryName>` function for every query that satisfies
**all** of:

1. It is an `INSERT` statement (sqlc sets `InsertIntoTable` internally).
2. Its command annotation is `:exec`, `:execresult`, or `:execrows`.
3. It has at least one parameter (`?` placeholder).

SELECT, UPDATE, DELETE queries and INSERT queries without parameters are silently
skipped.

---

## Generated code

Given the query:

```sql
-- name: InsertUser :exec
INSERT INTO users (name, email) VALUES (?, ?);
```

sqlc's regular code-gen produces `InsertUserParams` and `InsertUser`.
This plugin **additionally** generates:

```go
// Code generated by sqlc-gen-bulk-insert. DO NOT EDIT.

package db

import (
    "context"
    "strings"
)

const bulkInsertUserPrefix = `INSERT INTO users (name, email) VALUES `

// BulkInsertUser performs a bulk INSERT of multiple rows in a single statement.
// Passing an empty slice is a no-op.
func (q *Queries) BulkInsertUser(ctx context.Context, args []InsertUserParams) error {
    if len(args) == 0 {
        return nil
    }
    valueStrings := make([]string, 0, len(args))
    valueArgs := make([]any, 0, len(args)*2)
    for _, arg := range args {
        valueStrings = append(valueStrings, "(?, ?)")
        valueArgs = append(valueArgs, arg.Name, arg.Email)
    }
    query := bulkInsertUserPrefix + strings.Join(valueStrings, ", ")
    _, err := q.db.ExecContext(ctx, query, valueArgs...)
    return err
}
```

For queries with **exactly one** parameter, the function takes a `[]<GoType>`
slice instead of a `[]<QueryName>Params` struct slice:

```sql
-- name: InsertAuditLog :exec
INSERT INTO audit_log (user_id) VALUES (?);
```

```go
func (q *Queries) BulkInsertAuditLog(ctx context.Context, args []int64) error { … }
```

## MySQL type → Go type mapping

| MySQL type | Go type (NOT NULL) | Go type (nullable) |
|---|---|---|
| `tinyint(1)` / `bool` / `boolean` | `bool` | `sql.NullBool` |
| `tinyint` | `int8` | `sql.NullInt16` |
| `smallint` | `int16` | `sql.NullInt16` |
| `mediumint` / `int` / `integer` | `int32` | `sql.NullInt32` |
| `bigint` | `int64` | `sql.NullInt64` |
| `float` | `float32` | `sql.NullFloat64` |
| `double` / `real` | `float64` | `sql.NullFloat64` |
| `decimal` / `numeric` | `string` | `sql.NullString` |
| `char` / `varchar` / `text*` | `string` | `sql.NullString` |
| `binary` / `varbinary` / `blob*` | `[]byte` | `[]byte` |
| `date` / `datetime` / `timestamp` | `time.Time` | `sql.NullTime` |
| `json` | `json.RawMessage` | `json.RawMessage` |
| unsigned modifier | `u`-prefixed type (e.g. `uint64`) | — |

## Caveats

* The plugin only supports `?` positional placeholders (MySQL/MariaDB style).
  It is not suitable for PostgreSQL (`$1`, `$2`, …) or named parameters.
* The generated functions call `q.db.ExecContext` directly, which requires that
  your `Queries` struct uses the standard `db DBTX` field emitted by
  `sqlc-gen-go`.  Custom db-layer wrappers may need adaptation.
* Very large batch sizes can exceed MySQL's `max_allowed_packet`.  Consider
  chunking on the caller side when inserting thousands of rows.
